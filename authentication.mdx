## Overview

Kayel Calleja provides multiple authentication methods to secure your access to the platform. Choose API keys for simple server-to-server communication, OAuth for third-party applications, or enable two-factor authentication (2FA) for enhanced account security. Permissions follow role-based access control (RBAC) to manage team access effectively.

<Callout kind="alert">
  Always store credentials securely and never commit them to version control. Rotate keys regularly and use the principle of least privilege.
</Callout>

## Authentication Methods

Select the method that fits your use case. Use API keys for direct API calls, OAuth for user-authorized apps, or personal access tokens for Git-like operations.

<Tabs>
  <Tab title="API Keys" icon="key">
    Generate API keys from your dashboard for programmatic access.

    <Steps>
      <Step title="Generate Key" icon="settings">
        Navigate to `https://dashboard.example.com/account/api-keys` and click **Create New Key**.
      </Step>
      <Step title="Copy Key" icon="copy">
        Copy the generated key immediately—it displays only once.
      </Step>
      <Step title="Use in Requests" icon="code">
        Include the key in the `Authorization` header.
      </Step>
    </Steps>

    <ParamField header="Authorization" param-type="string" required="true">
      Bearer token format: `Bearer YOUR_API_KEY`.
    </ParamField>
  </Tab>

  <Tab title="OAuth 2.0" icon="shield">
    Integrate OAuth for apps that need user-specific access.

    <CodeGroup tabs="JavaScript,Python">
      ```javascript
      const response = await fetch('https://api.example.com/user', {
        headers: {
          Authorization: `Bearer ${accessToken}`
        }
      });
      ```
      ```python
      import requests
      response = requests.get(
        'https://api.example.com/user',
        headers={'Authorization': f'Bearer {access_token}'}
      )
      ```
    </CodeGroup>
  </Tab>

  <Tab title="Personal Access Tokens" icon="user">
    Use tokens for fine-grained permissions similar to GitHub.

    <ParamField query="token" param-type="string" required="true">
      Pass as query parameter for non-header auth.
    </ParamField>
  </Tab>
</Tabs>

## Two-Factor Authentication Setup

Enable 2FA to protect your account from unauthorized access.

<Steps>
  <Step title="Enable 2FA" icon="lock">
    Go to account settings at `https://dashboard.example.com/security/2fa`.
  </Step>
  <Step title="Scan QR Code" icon="smartphone">
    Use your authenticator app to scan the QR code.
  </Step>
  <Step title="Verify" icon="check-circle">
    Enter the 6-digit code to confirm setup.
  </Step>
</Steps>

<Callout kind="tip">
  Backup codes appear during setup—store them securely offline.
</Callout>

## Role-Based Access Control

Manage permissions with predefined roles: Admin, Developer, Viewer.

| Role     | API Access | Team Management | Dashboard View |
|----------|------------|-----------------|----------------|
| Admin    | Full       | Yes             | Full           |
| Developer| Scoped     | No              | Partial        |
| Viewer   | Read-only  | No              | Read-only      |

Assign roles via the team dashboard.

## Advanced Configuration and Troubleshooting

<ExpandableGroup>
  <Expandable title="OAuth Scopes" default-open="true">
    Define scopes like `read:user` or `write:repo` during authorization.

    ```javascript
    const scopes = ['read:user', 'write:ai'];
    // Redirect to: https://auth.example.com/oauth/authorize?scope=${scopes.join('%20')}
    ```
  </Expandable>

  <Expandable title="Common Issues">
    **Invalid Token (401):** Verify key format and expiration.

    **Permission Denied (403):** Check RBAC role.

    **Clock Skew (2FA):** Sync device time or regenerate codes.
  </Expandable>
</ExpandableGroup>
